MVP: Plataforma de Detección de Estrés y Generación de Material Educativo Accesible
Objetivo del MVP
El objetivo de este MVP es crear una plataforma capaz de detectar el estrés en estudiantes mediante un check-in rápido y datos biométricos de un smartwatch (frecuencia cardíaca, variabilidad de la frecuencia cardíaca - HRV, actividad física). La plataforma también debe generar material educativo accesible (resúmenes, lectura fácil, flashcards, quiz) adaptado a los niveles de estrés del usuario y mostrar gráficas de evolución del estrés.
________________


Requisitos del MVP
Funcionalidades principales
1. Detección de estrés mediante dos componentes:
   * Check-in de estrés (respuestas rápidas de bienestar, sueño, concentración).
   * Datos biométricos obtenidos de un smartwatch (frecuencia cardíaca, HRV, actividad física).
2. Generación de material educativo accesible, que se adapta al nivel de estrés:
   * Bajo estrés: resúmenes más largos y detallados.
   * Estrés medio: resúmenes simplificados y preguntas de quiz.
   * Alto estrés: micro-resúmenes y ejercicios breves.
3. Visualización de la evolución del estrés a través de gráficos generados con Plotly o Chart.js.
________________


Stack Tecnológico
El stack tecnológico para el MVP de 48 horas se ha diseñado para ser ágil y adecuado para la entrega rápida de un prototipo funcional.
Backend: FastAPI
* FastAPI será el marco principal para la API RESTful, encargada de recibir los datos del usuario, procesar el análisis de estrés y generar las respuestas adecuadas.
* Se utilizará Uvicorn como servidor ASGI para la ejecución local de la aplicación.
Frontend: Next.js + TailwindCSS + Chakra UI
* Next.js para la creación de la interfaz de usuario (UI) y la integración de las APIs del backend.
* TailwindCSS para un diseño responsivo y de alta calidad sin complicaciones.
* Chakra UI para componentes accesibles y estilizados que mejorarán la experiencia del usuario.
Gráficas: Plotly / Chart.js
* Plotly o Chart.js se utilizarán para la creación de gráficos interactivos y visualizaciones de la evolución del estrés en función del tiempo, utilizando los datos biométricos del smartwatch y los resultados del check-in.
Base de Datos: SQLite
* SQLite se utilizará como base de datos local para almacenar datos de usuario, historial de estrés y progreso.
Autenticación: Next.js Authentication (JWT)
* Se implementará la autenticación JWT para gestionar el inicio de sesión y la seguridad de las sesiones del usuario.
Smartwatch SDK: Health Connect (Android) / HealthKit (iOS)
* El SDK de Health Connect (para dispositivos Android) o HealthKit (para dispositivos iOS) permitirá acceder a los datos biométricos del smartwatch, como la frecuencia cardíaca, HRV, y actividad física del usuario.
IA: Hugging Face Transformers (GPT-2, DistilBERT)
* GPT-2 o DistilBERT de Hugging Face se utilizarán para generar resúmenes, simplificación de texto y creación de flashcards adaptados a los niveles de estrés del usuario.
Despliegue: vercel
* La aplicación se desplegará en vercel para facilitar el acceso y garantizar que el MVP esté disponible públicamente.

Plataformas Similares Existentes:
1. Headspace        
   * Propósito: Ofrece herramientas de meditación y mindfulness para reducir el estrés y la ansiedad.
   * Enfoque: Se centra principalmente en la meditación y el bienestar emocional, pero no adapta el contenido educativo ni utiliza datos biométricos para personalizar la experiencia de aprendizaje.

   2. Calm
   * Propósito: Aplicación de relajación y meditación que ayuda a los usuarios a reducir el estrés.
   * Enfoque: Similar a Headspace, pero más centrado en la relajación y el sueño, sin adaptación específica al entorno educativo ni conexión con datos biométricos.
   3. Forest
   * Propósito: Ayuda a los usuarios a enfocarse y reducir la procrastinación a través de la creación de "bosques virtuales".
   * Enfoque: Aunque motiva a los estudiantes a enfocarse, no personaliza el contenido educativo ni monitorea el estrés de manera integral.

      4. MindMeister
      * Propósito: Herramienta de mapas mentales para organizar el aprendizaje.
      * Enfoque: Se centra en la productividad, pero no adapta el contenido ni toma en cuenta el bienestar del estudiante mientras aprende.
